# playground 조사 요약

## 가져온 문제

- 현재 코드/API/문서 소유자 확인이 늦다.
- 이전 대화, 기억 자료, 오래된 문서가 현재 소유자보다 앞선다.
- 읽기 전용, 조사만, 수정 금지 같은 범위 지시가 선호처럼 처리된다.
- 레거시와 낡은 내용을 실제 현재 소유자 위임 없이 현재 근거처럼 남긴다.
- 근거보다 결론이 앞서고, 이후 근거가 결론을 방어하는 재료가 된다.

## source-owner-audit에 해당하는 부분

- 감사 시작 시 실행 경계 고정
- 현재 소유 표면 선확인
- 기억/과거 문서/후보 구현보다 현재 소스를 우선하는 순서
- 레거시, 낡은 문서, 대체 경로의 근거 전용 판정

## source-owner-audit가 맡지 않을 부분

- `BRIEF.md`와 logs의 저장 경계: `project-context`
- 문서 재구성 실행과 삭제 지시 반영: `structure-first-docs`
- 항상 읽히는 리포 지침: 각 리포 `AGENTS.md`

## 검토 기준

새 문구가 필요하다면 감사 초기에 무엇을 먼저 고정할지 분명히 해야 한다. 이미 있는 원칙을 길게 반복하거나 구현 금지로 과하게 읽힌다면 보류하는 편이 낫다.
